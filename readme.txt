Must Watch Movies Ontwerp documentatie

Casper Oostland - 500741279 - Project Interaction

Voor het vak front end voor designers heb ik een website ontworpen waar je informatie over films kan vinden die je gekeken moet hebben. Op de banner kun je nieuwe releases zien. De informatie over de films worden ingeladen met API. De website is responsive en werkt op de meeste devices. De banner kan worden bediend met de pijltjes toetsen en er zit een fade in op zodat de overgang niet te heftig is. Om de informatie over de film in te laden heb ik gebruik gemaakt van progressive disclosure om zo niet onnodig veel informatie te tonen zonder dat de gebruiker daar om gevraagd heeft.

Waar ben ik trots op? Als je de website laad, dan komt er een animatie naar voren die je verteld hoe je de slider moet bedienen.

Link naar opdracht 1, 2 en 3: https://casperoostland.github.io

Feedback uit de testen Na het testen van mijn website kreeg ik als feedback dat het niet duidelijk was dat je de slider kon bedienen met het toetsenbord. Ook was de overgang te hard als je van slider foto wisselde. Het design van kleuren vonden mijn test personen ook niet passen.


Principles of User Interface Design

Progressive disclosure laat alleen het nodige zien.  Vermijd een interface dat te veel tegelijk laat zien 
Het eerste gebruik is erg belangrijk bij het bezoeken van een website. Daarom kun je met een zero state de gebruiker uitleg geven over wat je zou kunnen doen, zo geef je sturing en kunnen gebruikers je site beter gebruiken.

A crucial moment: the zero state
Mensen vinden het fijn als ze het gevoel hebben van controle. Geef mensen het gevoel van controle door ze informatie te geven over de systeem status. (Als je dit doet dan gebeurd dat)
